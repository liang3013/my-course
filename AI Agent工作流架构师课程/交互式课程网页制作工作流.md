# 交互式课程网页制作工作流

> 基于 AI Agent 工作流架构师课程的制作经验梳理 · 2026-05-20

## 🎯 核心理念

**零依赖 · 单文件 · 即开即用**

一个 `.html` 就是一门完整课程。用户双击打开就能学，不需要服务器、不需要 npm install、不需要任何外部资源。

### 为什么选这个路线

| 方案 | 上手 | 分发 | 交互 | 结论 |
|------|:--:|:--:|:--:|------|
| Notion/飞书 | 快 | 🔗链接 | 弱 | 不像正经课程 |
| PPT/PDF | 快 | 📎附件 | 无 | 没法做测验 |
| React/Vue SPA | 慢 | 需部署 | 强 | 太重 |
| **纯 HTML 单文件** | 中 | ✅双击 | 强 | ✅ 最优 |

## 🎨 SVG 图表制作流程

### 1. ASCII 原型（快速构思）

先用 ASCII 画布确定结构和布局：

```
┌──────────┐     ┌──────────┐
│  输入    │ ──▶ │  处理    │ ──▶ 输出
└──────────┘     └──────────┘
```

### 2. 转换为内联 SVG

用 `viewBox` 控制缩放，`<rect>`/`<circle>`/`<line>`/`<text>` 代替 ASCII 字符。

```svg
<svg viewBox="0 0 800 200" xmlns="http://www.w3.org/2000/svg">
  <rect x="20" y="30" width="160" height="50" rx="8" fill="var(--accent)" />
  <text x="100" y="60" text-anchor="middle" fill="#fff">输入</text>
  <!-- ... -->
</svg>
```

### 3. 嵌入课程

包裹在 `.diagram-svg` 容器中：

```html
<div class="diagram-svg">
  <svg viewBox="0 0 800 200">...</svg>
</div>
```

### 4. 关键规则

| 规则 | 做法 | 原因 |
|------|------|------|
| **字体继承** | SVG `<text>` 不写 `font-family`，用 CSS `.diagram-svg text { font-family: inherit; }` | 全局统一、免逐个修改 |
| **颜色用 CSS 变量** | `fill="var(--accent)"` 而非硬编码色值 | 换主题自动适配 |
| **viewBox 设宽高比** | `viewBox="0 0 800 200"` | 换窗口大小不变形 |
| **字号层级** | 粗体标题 10 / 描述 9 / 装饰图标 14 | 约页内文本 0.7×，视觉不抢 |
| **背景统一** | `.diagram-svg { background: var(--card); }` | 与卡片同色，不割裂 |

### 注意

- `fill="var(--xxx)"` 在部分浏览器渲染器中 CSS 变量解析不稳定，复杂图可保留硬编码色值作为备选
- SVG 中的 emoji 渲染效果因系统而异，推荐用简单几何图案代替

## 🎨 CSS 架构

### 变量体系（每文件独立调色）

```css
:root {
  --bg: #0d1117;        /* 页面底色 */
  --card: #161b22;      /* 卡片底色 */
  --card-hover: #1c2333;/* 卡片悬停 */
  --border: #30363d;    /* 边框/分割线 */
  --text: #e6edf3;      /* 正文 */
  --sub: #8b949e;       /* 次要文字 */
  --accent: #58a6ff;    /* 主强调色（蓝） */
  --accent2: #3fb950;   /* 副强调（绿） */
  --accent3: #d2a8ff;   /* 第三强调（紫） */
  --red: #f85149;
  --yellow: #d2991d;
  --radius: 12px;
}
```

**四门课的四套配色**：

| 课程 | --accent | --bg | 色系风格 |
|------|----------|------|----------|
| 🟢 初级 | `#58a6ff` 蓝 | `#0d1117` | 蓝紫青 |
| 🟡 中级 | `#58a6ff` 蓝 | `#0d1117` | 紫蓝绿 |
| 🟠 高级 | `#f0a040` 橙 | `#0a0e14` | 橙蓝绿 |
| 🔴 专家 | `#e86040` 红 | `#06080d` | 红蓝金 |

### 类命名规则

| 组件 | 初/中级 | 高/专家 | 原因 |
|------|---------|---------|------|
| 导航项 | `.nav-item` | `.nav-i` | 不同作者命名习惯 |
| 激活态 | `.nav-item.active` | `.nav-i.act` | 同上 |
| 课程卡片 | `.lesson-card` | `.lesson` | 同上 |
| 测验 | `.quiz-card` | `.quiz` | 同上 |
| 按钮主色 | `.btn-primary` | `.btn-p` | 同上 |
| 二级按钮 | `.btn-secondary` | `.btn-s` | 同上 |
| 成功按钮 | `.btn-success` | `.btn-go` | 同上 |
| 步骤卡片 | `.step-card` | `.step` | 同上 |
| 证书 | `.certificate` | `.cert` | 同上 |

> ⚠️ 三套命名不是设计意图，是同一模板在不同时间构建的结果。新项目应统一命名。

## ✨ 视觉美化清单

当拿到一个裸功能可用的课程 HTML 后，按以下清单逐项打磨（这些是 index.html→三文件同步的完整 checklist）：

### 全局

- [ ] `html { scroll-behavior: smooth; }`
- [ ] `::selection` 覆盖为半透明主题色
- [ ] WebKit 自定义滚动条（6px 宽，圆角滑块）
- [ ] `body { background: radial-gradient(...) }` 径向渐变（顶部更亮）
- [ ] `body { -webkit-font-smoothing: antialiased; }`

### 侧边栏

- [ ] 导航项左边透明 3px 条 `.nav-item { border-left: 3px solid transparent; }`
- [ ] 悬停时左边条半透明 `:hover { border-left-color: rgba(accent,0.3); }`
- [ ] 激活时左边条实色 `:border-left-color: var(--accent);`
- [ ] 进度条 glow 阴影 `box-shadow: 0 0 12px rgba(accent,0.2);`
- [ ] 进度条 transition 用 cubic-bezier

### 内容卡片

- [ ] 卡片 hover 微光阴影 `.lesson-card:hover { box-shadow: 0 2px 20px rgba(accent,0.05); }`
- [ ] 标题区 meta 下方加分隔线 `border-bottom: 1px solid var(--border)`
- [ ] h3 左侧渐变竖线 `h3::before { ... 3px gradient bar ... }`
- [ ] 切课页淡入动画 `@keyframes fadeIn`

### 类比/提示框（analogy / tip / warning / deep / decision）

- [ ] 左边色条 `border-left: 3px solid var(--xxx)`
- [ ] 背景透明度降低（0.07→0.05）不抢眼
- [ ] 标签加 flex 对齐 + gap 间距
- [ ] hover 时边框加深

### 测验

- [ ] 边框改左边条（非全框）`.quiz-card { border-left: 3px solid var(--accent); }`
- [ ] 选项 hover 右移 `transform: translateX(4px)`
- [ ] 正确/错误状态也加左边色条
- [ ] 禁用态取消 transform

### 按钮

- [ ] 主按钮加阴影 `box-shadow: 0 2px 8px rgba(accent,0.25)`
- [ ] hover 时阴影加深 `box-shadow: 0 4px 16px rgba(accent,0.35)`
- [ ] active 缩放 `transform: scale(0.97)`
- [ ] display 改为 inline-flex + gap

### 步骤卡片

- [ ] 序号圆标加大（32→36px）+ 阴影
- [ ] hover 上浮 `transform: translateY(-2px)`
- [ ] 描述行高加大 `line-height: 1.6`

### 表格

- [ ] 外层包 `.table-wrap`（`overflow-x:auto` + `border-radius` + `border`）
- [ ] th 背景改半透明主题色 `rgba(accent,0.06)` + 大写 + 字母间距
- [ ] th 文字颜色用 `var(--accent)`
- [ ] `tr:last-child td { border-bottom: none; }`
- [ ] 行 hover 微亮 `tr:hover td { background: rgba(accent,0.03); }`

### 证书

- [ ] 分数大号渐变文字 `background: linear-gradient(...); -webkit-background-clip: text; -webkit-text-fill-color: transparent;`
- [ ] 外围发光 `box-shadow: 0 4px 32px rgba(accent,0.06)`
- [ ] 显示时加 fadeIn 动画
- [ ] padding 加大（40→48px）

## 🧩 交互式 JavaScript 模式

### 导航系统

```js
// 核心：点击导航 → 切课 → 更新 UI
function navigateTo(index) {
  currentLesson = index;
  renderContent(lessons[currentLesson]);
  updateUI();
  save();  // localStorage
}
```

### 进度持久化

```js
function save() {
  const data = { currentLesson, completed: [...completedSet], quizScores };
  localStorage.setItem('agent_course_progress', JSON.stringify(data));
}
```

单选答案验证 + 多选不覆盖 + 正确/错误立即显示解释：

function answerQuiz(btn, questionIndex, chosenIndex) {

## 🔄 跨文件统一美化流程

当需要对多个同类文件统一做视觉升级时：

### Step 1：选一个做标杆

选类名最规范的那个文件（本例是 index.html），先做到满意。

### Step 2：对比差异矩阵

列出每个文件的差异点：类名映射 + 色值 + 是否有特殊组件。

| 对比维度 | index | intermediate | advanced | expert |
|----------|:--:|:--:|:--:|:--:|
| 类名规范 | ✅ | ✅ | ❌ 缩略 | ❌ 缩略 |
| 特殊组件 | — | — | 决策树/拖拽/填空 | 对比框/矩阵/决策 |
| 提示框类型 | tip/warn/deep | ←同 | ←同 | +decision |

### Step 3：逐一适配

用 `edit` 工具逐文件、逐组件替换 CSS 块。每个 edit 只改一个组件类型（如"所有按钮"），改完验证：

```bash
grep -c "关键CSS属性" 文件名.html  # 对比标杆数量
```

### Step 4：特殊组件不强行统一

高级/专家独有的交互组件（决策树/拖拽/对比框）不套用普通组件的美化规则，保持功能性。

## ⚠️ 踩过的坑

| 坑 | 表现 | 解法 |
|----|------|------|
| **类名不一致** | 同一个组件在四个文件有 .nav-item / .nav-i / .lesson-card / .lesson 等多套命名 | 建立类名映射表再动手，别假设通用 |
| **edit 工具需精确匹配** | oldText 稍有差异就报 "Could not find exact text" | 先用 read 读原文，再抄写做 oldText |
| **CSS 变量在 SVG 中不稳定** | `fill="var(--accent)"` 部分浏览器渲染异常 | 复杂 SVG 用硬编码色值；简单 SVG 可用变量 |
| **单行 vs 多行 CSS** | advanced/expert 用单行紧凑写法，index 用多行展开 | edit 时注意匹配原始风格 |
| **emoji 在 SVG 中渲染不一致** | 不同系统显示效果不同 | 用几何形状代替，或用字号较大的 text 元素 |
| **雪球 ProseMirror 粘贴不稳定** | paste 后内容被清空 | 等编辑器聚焦后再 paste，同一会话内重试 |
