## 📝 内容设计模式

### 1. 类比（analogy）——最重要的设计元素

每个抽象概念必须配一个**生活化类比**。原则：**通俗性 > 专业性**。

```html
<div class="analogy">
  <div class="label">🏪 盘下小卖部</div>
  <p>你花30万盘下街角小卖部。货架冰柜只值10万。多付的20万 = 商誉。</p>
</div>
```

#### 类比选型铁律

| ❌ 失败的类比 | ✅ 成功的类比 | 为什么 |
|------|------|------|
| 🏥 体检三件套（血常规/CT/心电图）| 🚗 买二手车（看照片/查档案/试驾）| 买车人人有经验；医院检查抽象 |
| 📱 抖音号被封→商誉归零 | 🏪 小卖部隔壁开沃尔玛→商誉减值 | 抖音号被封连基础资产都没了；小卖部类比精准对应商誉减值本质 |
| 任何需要解释"为什么"的类比 | 一说完立刻能脑补画面的类比 | 类比的作用是瞬间理解，不是增加新概念 |

#### 类比设计检查清单

- [ ] 三秒内能脑补画面？
- [ ] 不需要额外解释"类比本身"？
- [ ] 精准对应概念的核心机制（不是表面相似）？
- [ ] 类比的"失败"条件和概念的"失效"条件一致？

### 2. SVG 图表——金融课程特殊规则

金融图表数字多、公式多，SVG 字号必须比技术课程大：

| 元素 | AI Agent 模板 | 金融课程 |
|------|:--:|:--:|
| 粗体标题 | 10px | 14-15px |
| 描述文字 | 9px | 12px |
| 公式/数字 | 11px | 13-14px |
| 次要标注 | 8px | 11px |

#### SVG 间距铁律：先松后调

```
❌ 做法：先画紧凑 → 发现重合 → 逐个微调 → 越调越乱
✅ 做法：viewBox 先设大（820×500）→ 每级留 24px+ 间隙 → 验证后再收窄
```

#### SVG 布局三步法

1. **先列元素清单**：几个主块、几个箭头、几个标注、几个注释框
2. **分配空间**：主流程居中（Y轴均分）、减项标签统一右列、注释框与对应层级水平对齐
3. **验证不重叠**：逐对检查相邻元素的 Y 坐标差 ≥ 元素高度 + 4px

#### 多层 SVG 的标准间距表

| 场景 | 间距 | 说明 |
|------|:--:|------|
| 主块高度 | 36-44px | 标题+一行副文本 |
| 箭头段 | 20-28px | 含减项标签文字（12px字号） |
| 两个主块之间 | 块高 + 箭头段 | 约 56-72px |
| 右侧注释框 | 与对应主块同 Y | 严格水平对齐 |
| 左侧案例区 | 独立 X 区域 | 不干扰主流程 |
| viewBox 高度 | 元素总数×70 + 100 | 预留底部总结行 |

## 🧩 互动组件模式

### DCF 计算器（模块八）

```html
<!-- HTML：内联 oninput 直接调用全局函数 -->
<input type="number" id="dcf-fcf" value="100" oninput="calcDCF()">
<input type="number" id="dcf-g1" value="15" oninput="calcDCF()">
<input type="number" id="dcf-g2" value="3" oninput="calcDCF()">
<input type="number" id="dcf-wacc" value="10" oninput="calcDCF()">
<div id="dcf-val">0 亿元</div>
```

```js
// JS：全局函数 + renderContent 中延迟初始化
window.calcDCF = function() {
  var f = parseFloat(document.getElementById('dcf-fcf').value) || 0;
  // ... 计算逻辑 ...
  document.getElementById('dcf-val').textContent = pv.toFixed(0) + ' 亿元';
};

// 在 renderContent 的 setTimeout 中初始化
if (id === 'dcf-basics') {
  setTimeout(function() { if (window.calcDCF) calcDCF(); }, 50);
}
```

### ROE 三滑块（模块六）

```html
<input type="range" min="2" max="50" value="20" id="roe-npm" oninput="updateROE()">
```

模式同 DCF：HTML 内联 oninput → JS 全局函数 → renderContent 延迟初始化。

### 现金流 8 模式选择器（模块五）

复杂 UI 用 `setTimeout` + 动态生成 DOM：

```js
// 在 renderContent 中
if (id === 'cashflow-patterns') {
  setTimeout(function() {
    var cfModes = [/* 8种模式数据 */];
    var html = '';
    cfModes.forEach(function(m, i) {
      html += '<div onclick="selectCFMode(' + i + ')">...';
    });
    document.getElementById('cfSelector').innerHTML = html;
    selectCFMode(0);
  }, 50);
}
```

**关键点**：互动组件的数据和 JS 逻辑**永远不要放在 lesson content 的模板字面量里**，而是放在主 script 块的全局作用域，content 只放静态 HTML 骨架。

## ⚠️ 本课程踩过的坑（比模板工作流的多）

### 🔴 致命级

| 坑 | 表现 | 根因 | 解法 |
|----|------|------|------|
| **模板字面量内嵌 `<script>` 标签** | 页面完全空白，零 JS 报错 | HTML 解析器在遇到 `</script>` 时立即关闭主 script 标签——不管它在不在 JS 字符串里 | ❌ 绝不在 lesson content 中写 `<script>` 标签 ✅ 互动组件用全局函数 + setTimeout 初始化 |
| **quiz explain 中的中文引号** | `SyntaxError: Unexpected identifier` | JS 单引号字符串中含有未转义的单引号字符 | 用 `\'` 转义，或改用双引号包裹 explain 字符串 |

### 🟠 严重级

| 坑 | 表现 | 根因 | 解法 |
|----|------|------|------|
| **SVG 图表元素重叠** | 文字被遮挡、标注看不清 | 间距太紧（Y 轴差<20px），减项标签与右侧注释框挤在同一区域 | viewBox 先放大到 820×500 → 每级间距≥24px → 左右注释严格水平对齐 |
| **SVG 减项标签散落** | 五个减项（-成本、-费用、±其他、-税）在五个不同 X 坐标 | 没有统一规划右列位置 | 所有减项标签用同一个 X 坐标（如 x=530），形成整齐的右对齐列 |
| **测验题目指代不清** | 「三张报表」但没有列是哪三张 | 假设学员已经记得 | 题目中显式列出：「利润表、资产负债表、现金流量表」 |

### 🟡 中等级

| 坑 | 表现 | 根因 | 解法 |
|----|------|------|------|
| **类比不贴切** | 抖音号被封=商誉归零——但抖音号封了连基础价值也归零了 | 类比的失败机制和概念不匹配 | 换成小卖部隔壁开沃尔玛——冰柜货架还在，只是预期溢价没了 |
| **ROE 图权益乘数放第二行标「隐含在上方」** | 概念错误——权益乘数是独立因子，不是隐含的 | 对杜邦公式理解不够 | 三因子同排一行：净利率 × 周转率 × 权益乘数 |
| **利润表阶梯图机构乱** | 5级显示但右栏注释位置不齐 | 第一版急于出图，没规划布局 | 重画：左案例区+中主流程+右注释+底总结行 |

### 标准模块（含课时+测验）

在 `courseData` 数组末尾（`];` 前）插入：

```js
      },
      {
        id: 'new-lesson-2',
        title: '第二课',
        emoji: '📊',
        content: `HTML内容...`
      },
      {
        id: 'quiz-new',
        title: '🧪 小测验',
        emoji: '🧪',
        quiz: {
          questions: [
            {
              q: '题目？',
              options: ['A', 'B', 'C', 'D'],
              correct: 0,
              explain: '解释...'
            }
          ]
        }
      }
    ]
  },
```

### 实战案例模块

模式同上，但内容设计遵循两个原则：

1. **全程与已学模块对照**：利润表→对照模块三，资产负债表→对照模块四，现金流→对照模块五，ROE→对照模块六
2. **最好有两个案例对比**：茅台 vs 美的——同一套杜邦公式，两种完全不同的赚钱方式

案例测验设计要点：
- 不是考记忆，是**考理解和判断**
- 包含计算题（如"净利率涨3pp→ROE变多少"）
- 包含判断推理题（如"商誉290亿该如何评估"）
- 最后一道题收尾升华（如"谁更适合初学者"→回归能力圈概念）
